# Server接入

提示

appId, ip地址、端口、appSecret、appKey参数 ，请查看我方提供的README.txt文件

appSecret: 用于支付、发货相关的接口参数拼接签名

appKey: 用于非支付相关的接口参数拼接签名

# Token校验接口【必接】

注册登录流程会进行Token校验，详见 业务流程 - 登录流程

  • 请求URL地址

    http://ip地址:端口/mythsgame/user/verifyToken

  • 请求参数

    HTTP协议，支持post方法

    参数名称类型示例值必须描述
    appIdint是应用ID,由平台上分配
    userIdint是平台用户ID
    tokenString是用户校验的token
    signString是签名，md5(按照参数key字段升序对应的值组成原串 + appKey)

  • 返回参数

    以JOSN格式返回结果

    名称类型示例值描述
    codeint返回状态 200=成功,其他等于失败
    error_msgString错误信息

# 接收充值结果接口，完成发货 (CP方提供)【必接】

用于支付流程， 详见 业务流程 - 支付流程

  • 请求URL地址 （由CP提供）

注意

网页充值 type=1，不会产生gameOrderId，只需对roleId进行合法性校验;
游戏包充值 type=2，需要增加对gameOrderId的合法性校验。

保险起见，游戏方拿到productName参数，进行游戏内的商品逻辑判断，然后读取自己的商品配置表，获取对应的商品信息，如钻石数，礼包内容等进行发货处理。

gameCoinCurrency=Diamond时，为虚拟币充值，此时gameCoin为钻石数，不建议CP取此数据直接进行发货。
gameCoinCurrency≠Diamond时，为礼包等特殊商品或礼包购买，此时gameCoin的值无效。

  • 请求参数

    HTTP协议，支持POST方法

    参数名称类型示例值必须描述
    transactionIDString是交易流水，由平台商产生，唯一
    appIdint是应用ID,由平台上分配
    userIdint是平台用户ID
    roleIdString是游戏角色ID
    gameOrderIdString是游戏订单ID
    gameZoneIdString是游戏区服ID
    typeint是0 = 游戏包充值 1= WEB充值
    productNameString是商品名称
    productNumint是商品数量
    amountdouble是商品金额，如9.99
    currencyString是商品货币，如USD
    gameCoinint是游戏币数量
    gameCoinCurrencyString是Diamond 或 礼包ID，当为Diamond时，为虚拟币充值；为其他值时，是某类礼包购买。
    gameExInfoString是扩展字段，下单时传入，原样返回
    extString是额外字段
    signString是md5(按照参数key字段升序对应的值组成原串 + appSecret)

  • 返回参数

    以JOSN格式返回结果

    名称类型示例值描述
    codeint返回状态 200=成功,其他等于失败
    error_msgString错误信息

# 获取角色信息接口 (CP方提供)【必接】

  • 请求URL地址 （由CP提供）

  • 请求参数

    HTTP协议，支持GET方法

    参数名称类型示例值必须描述
    timestampString是时间戳(System.currentTimeMillis())
    userIdint是平台ID
    gameZoneIdString是游戏区服ID
    signString是md5(按照参数key字段升序对应的值组成原串 + appKey)

  • 返回参数

    以JOSN格式返回结果

    名称类型示例值描述
    codeint返回状态 200=成功,其他等于失败
    dataObject
    data.rolesArray
    data.roles.roleIdString角色ID
    data.roles.roleNameString角色名称
    data.roles.roleLevelString角色等级
    data.roles.stateint0=正常 1=删除 2=废弃
    data.roles.createTimeString角色创建时间
    data.roles.lastLoginTimeString角色登录时间
    data.roles.lastPaymentTimeString角色最后支付时间
    data.roles.vipLevelStringvip等级
    data.roles.gameCoinTotalString剩余的元宝数

# 根据流水号查看订单状态接口（开发中...）

注意

此接口调整开发中，暂不对外提供

研发发货前可以通过此接口来确认订单是否成功

  • 请求URL地址

    http://ip地址:端口/mythsgame/client/order/confirm/{appId}/{transactionId}/{sign}

  • 请求参数

    HTTP协议，支持GET方法

    参数名称类型示例值必须描述
    appIdint是应用ID，由平台分配
    transactionIdString是交易ID
    signString是md5(按照参数key字段升序对应的值组成原串 + appSecret)

  • 返回参数

    以JOSN格式返回结果

    名称类型示例值描述
    codeint返回状态 200=成功,其他等于失败
    dataObject
    data.transactionIdString交易流水，平台商产生，唯一
    data.amountdouble交易金额
    data.currencyString交易货币
    data.gameCoinint游戏币数量
    data.userIdint平台用户ID
    data.timeString交易时间

SDK Server将通过此接口判断游戏内是否发货成功

  • 请求URL地址 （由CP提供）

  • 请求参数

    HTTP协议，支持GET方法

    参数名称类型示例值必须描述
    timestampString是时间戳(System.currentTimeMillis())
    userIdint是平台ID
    roleIdString是游戏角色ID
    gameZoneIdString是游戏区服id
    signString是md5(按照参数key字段升序对应的值组成原串 + appSecret)

  • 返回参数

    以JOSN格式返回结果

    dataObject
    data.gameOrderIdString游戏订单id

# sign签名

  • 对所有待签名参数按照字段名（key）的 ASCII 码从小到大升序排序（字典序）后，把键值对拼接（即key1=value1key2=value2…）成字符串 string1。这里需要注意的是参数名区分大小写。
  • 拼接API密钥（key1=value1key2=value2…appkey） 注意：部分需要拼接appkey，部分接口需要拼接appSecret
  • 签名 sign=MD5(string1).toLowerCase()="9a0a8659f005d6984697e2ca0a9cf3b7"
